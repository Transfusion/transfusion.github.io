ExPo2
A stock exchange simulator allowing humans and bots to trade together; a rewrite of Exchange Portal in Java. I extended the system with ØMQ and exposed an API for bot developers to enable simulations of high frequency trading.
ExPo2 in the state it was presented to me used the WebSocket protocol for order entry and market data for both the web trading interface and the bots. I developed a benchmarking harness to obtain latency figures at various load levels, added a protobuf representation of the messaging protocol since ØMQ supports binary messages, and added an endpoint where compiled bots could be uploaded and run inside the exchange, thus eliminating any latency. A load test using Apache JMeter was finally performed with up to 60 simultaneous threads.
InstaSketch
A sketch-based image retrieval Android app with a local mode meant to return closest matches in the user's photo album and a remote backend mode in Python, incorporating both color and spatial features.
Proof of concept that self-contained mobile CBIR systems are viable. I evaluated various color features such as color coherence vectors, tree-based methods, and color histograms, spatial features such as perceptual hashes, wavelet-based methods, and other descriptors invariant to various minor distortions such as SIFT/SURF applied to a bag of words model to find the relative frequency of keypoints against the corpus of images. Various distance measures and preprocessing strategies were compared for feature vectors of the same type. Unfortunately, only color histograms were implemented in the local mode due to time constraints, and the success of spatial features severely limited by the degree of precision of the touch input and the user's image in mind.
PlainShare
A node.js chatroom/file sharing application with persistence so users can see files and messages from before they joined. Uses socket.io for the realtime chatbox; backed by Redis for the chatbox/session management and message log.
The original plan was to use peer.js and skyway to broker direct peer to peer connections whenever possible for efficiency; caching files server-side was first implemented because one-to-many file transfer and NAT traversal were the biggest issues.
GeoGame
A sample location-based game utilizing all four Android components. Notifications are sent and new tasks generated whenever the user reaches the task's generated location.
A ContentProvider wrapping an associated SQLiteOpenHelper stores the player's trail and a LocalBroadcastReceiver sends location updates from the LocationManager to the Activity.
SketchAndGuess
A color-layout based image retrieval system which segmented images in the database and computed the median color for each segment in various color spaces such as HSV, RGB, and Lab, and constructed a tree for each segment, thus forming a random forest. I contributed to the development of a naïve bayesian approach which predicted the most probable color of the unfilled regions in the specified layout, based on any filled surrounding regions.
The system was a Java backend for an Android app; I added a Swing GUI for visualization and an SQLite database to store the image descriptors as blobs, a large speed improvement over the previous plaintext storage. I also parallelized the feature extraction process, both at the individual image level and on the separate segments of the image to achieve a speedup almost linear to the number of cores in the machine.
